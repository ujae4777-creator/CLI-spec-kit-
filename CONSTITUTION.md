# CONSTITUTION — 개발 헌법

> **① Constitution** 층 — "어떻게 짜나" (전 프로젝트 공통, 거의 불변).
> 모든 세션은 코드를 건드리기 전에 이 파일을 읽는다.
> 위반은 버그와 동급으로 취급한다. 작업 끝에 "어긴 규칙 없나" 셀프체크 후 보고한다.
> 실제 프로젝트에 넣을 땐 이 내용을 그 프로젝트의 `CLAUDE.md`로 복사한다(자동 로드용).

## ⚠️ 보안 경고 — 다른 모든 규칙보다 먼저

> 능력을 주기 전에 고삐부터 쥔다. 위반은 버그가 아니라 **사고**다.

- **신뢰 경계:** **무엇을 실행할지 결정하는 입력**(CLI 경로·명령·스크립트 위치·env 변수)은 **신뢰되지 않은 출처**(프로젝트 폴더의 `.env`, clone한 저장소, 외부 파일)에서 **절대 받지 않는다.**
- **allowlist > denylist:** 막을 것을 고르지(denylist) 말고, **허용할 것만 연다(allowlist).** 기본은 차단. — denylist는 빠뜨리면 뚫린다.
- **최소 권한:** 도구·MCP는 꼭 필요한 것만, 가능한 **읽기 전용**으로, 작업 후 회수. **비가역·외부 부작용**(실거래·송금·prod 변경·임의 셸 실행)은 자동 연결하지 않고 건마다 사람이 승인.
- **참고 사례:** Ouroboros CVE-2026-47211 — 프로젝트 폴더 `.env`를 신뢰해 실행 경로로 써서 RCE. denylist 1차 수정도 빠진 변수로 재차 뚫림(GHSA-JV2H-4P9V-WF5W).

## 0. 최우선 원칙
- **분명해지기 전엔 만들지 않는다.** 모호하면 코드 대신 질문하거나 `SPECIFY.md`를 먼저 갱신.
- **만든 건 다른 눈으로 검증한다.** (사람 실측 + 반대 모델 리뷰)
- **결정은 문서로 박제한다.** 구조 결정은 `PLAN.md`에 이유와 함께.
- **한 번에 작게 바꾼다.** 큰 변경은 diff 수준 계획을 먼저 제시하고 승인받는다. 한 커밋 = 한 가지 일.

## 1. 새로 만들기 전 — 중복 방지
- 새 function/helper/type/shape를 만들기 전에 **기존에 있는지 먼저 검색**한다. 있으면 재사용.
- 공용 유틸·타입·shape는 **SSOT(단일 출처)에서만** 정의·임포트한다. → `<SSOT 경로 지정>`
- "비슷한 게 이미 있나?"에 답하지 못하면 새로 만들지 않는다.

## 2. 구조·경계
- 의존은 **한 방향**으로만. **순환 의존 금지** (lint/CI 강제).
- **레이어/도메인 경계를 넘지 않는다.** 경계 정의는 `PLAN.md` 참조.
- **숨은 결합 금지**: 암묵 계약, 초기화 순서 의존, 전역 상태 공유, side-effect로 엮인 모듈. 의존은 **명시적으로**(인자/주입)로만 드러낸다.
- **re-export 방치 금지.** barrel/index는 의도가 있을 때만.

## 3. 복잡도
- **중첩 3단계 초과 금지.** early return / guard clause로 평탄화 (lint `max-depth` 강제).
- **갓펑션·갓파일 금지.** 함수는 한 가지 일만 (lint `complexity` 강제).
- 단, **라인 수 맞추려고 응집된 로직을 억지로 쪼개지 마라.** 복잡도 기준으로 나눈다 (라인 수는 경고일 뿐).

## 4. 에러
- **조용히 삼키지 마라.** 빈 `try/catch`, 무의미한 fallback, 호출부마다 방어코드 금지.
- 에러는 **경계/핸들러 한 곳에서만** 처리한다. 그 외엔 던진다 (**fail loud**).

## 5. 타입 (강제)
- **strict.** `as any`, `as unknown as` 금지 (lint 강제). JS면 JSDoc으로 계약 표현.
- 타입이 **암묵 계약을 드러내게** 한다.

## 6. 테스트
- **출력 행동을 검증한다.** 함수명 존재 여부 같은 문자열 확인 금지. 커버리지 숫자만 늘리지 마라.
- **엣지케이스 먼저**: 빈 입력, null, 경계값, 실패 경로, 동시성. 코드 전에 테스트로 고정.
- **mock는 외부 경계에서만.** 내부 구현 세부 mocking 금지.
- 테스트를 통과시키려고 **프로덕션 코드에 방어 코드를 더하지 마라.**

## 7. 마무리
- 기존 코드를 대체할 땐 **중복·죽은 코드·주석 처리 코드를 남기지 마라.**
- 작업 종료 시: 변경 요약 + 어긴 규칙 셀프체크 + 다음 검증 단계 제안.
