# CLARIFY — 기획 인터뷰 스킬

> **③ Clarify** 층 — Specify의 모호성을 질문으로 메운다.
> **발동:** "이 파일로 인터뷰 시작해줘" 또는 새 기획을 시작할 때.
> **목적:** 사용자가 빈 양식을 노려보지 않게 한다. AI가 묻고 추론해서 `SPECIFY.md`를 대신 채운다.
> **결과물:** 채워진 spec 파일 1개 (`[프로젝트명]_SPEC.md`).

---

## 핵심 원칙 (이걸 어기면 인터뷰가 망한다)

1. **초안 먼저, 질문은 나중.** 빈 칸을 사용자에게 떠넘기지 마라. 한두 문장만 받아도 §1~§8 초안을 **추론으로 통째 채운 뒤**, 확신 없는 곳만 물어라. 사용자는 창작이 아니라 **교정**만 한다.
2. **한 번에 1~2개만 묻는다.** 질문 폭탄 금지. 대화하듯 주고받는다.
3. **추론 가능한 건 묻지 않는다.** 문제를 들으면 해결방향·입출력·완료기준은 대부분 유추된다. 유추한 걸 "이렇게 이해했는데 맞아?"로 확인만 받아라.
4. **모호하면 멈춘다.** 모든 섹션이 채워지고 빈 칸·`TBD`가 0이 될 때까지 코드 얘기로 넘어가지 않는다. (= 명확성 게이트)
5. **사용자 말투로 채운다.** 사용자가 쓴 단어를 살려라. 매끄럽게 윤문한답시고 의미를 바꾸지 마라.

---

## 진행 절차

### STEP 0 — 씨앗 받기
사용자에게 딱 한 가지만 요청한다:
> "만들고 싶은 걸 1~2문장으로 던져줘. 완벽하지 않아도 돼."

### STEP 1 — 초안 통째로 채우기
받은 씨앗으로 `SPECIFY.md`의 §1~§8을 **전부 추론해서 채운다.**
- 확신 있는 칸 → 그냥 채움
- 추측한 칸 → 채우되 `🔸추측` 표시
- 정말 모르는 칸 → `❓` 표시

채운 초안 전체를 사용자에게 보여준다.

### STEP 2 — 좁혀 묻기 (반복)
`🔸추측`·`❓` 칸 중 **가장 중요한 1~2개만** 골라 질문한다.
- 우선순위: §1 한 줄 설명 > §2 문제 > §6 범위(제외) > §7 완료기준 > 나머지
- 사용자 답을 받으면 해당 칸 갱신 + 그 답에서 파생되는 다른 칸도 같이 갱신.
- 다음 1~2개로 넘어간다.

### STEP 3 — 게이트 점검
매 라운드 끝에 스스로 점검:
- [ ] §1 한 줄 설명이 그 자체로 이해되는가?
- [ ] §6 ✅필수와 ❌제외가 서로 모순 없는가?
- [ ] §7 완료기준으로 "끝났는지" 판정 가능한가?
- [ ] 남은 `🔸`·`❓`·`TBD`가 0인가?

하나라도 미달이면 STEP 2로 돌아간다. 다 통과하면 STEP 4.

### STEP 4 — 마감
- `🔸`·`❓` 표시를 모두 제거한 깨끗한 spec으로 정리.
- 프론트매터 `id` 생성, `sources`·`domain`은 사용자에게 확인.
- 본문 상단 `status: draft → specifying` 으로 갱신.
- `[프로젝트명]_SPEC.md`로 저장하고, 다음 단계(`PLAN.md` 작성 → 구현) 제안.

## 묻기 좋은 질문 (참고)

- "이거, 너 혼자 쓸 거야 남도 쓸 거야?" → 대상 사용자
- "지금은 이 일을 어떻게 하고 있어?" → 현재 방식 + 불편함
- "안 만들 기능 하나만 정하면?" → §6 제외 (범위 통제의 핵심)
- "어떤 상태가 되면 '됐다' 싶어?" → §9 완료기준
- "이거 만들 생각, 어디서 나왔어? (어떤 책·개념·문제)" → `sources` (지식 연결)

## 하지 말 것

- 18개 질문을 한꺼번에 나열하기
- 빈 양식을 그대로 사용자에게 던지고 "채워주세요" 하기
- 사용자가 안 말한 기능을 §6 필수에 멋대로 추가하기
- 모호한 채로 "일단 만들어보죠" 하고 넘어가기
